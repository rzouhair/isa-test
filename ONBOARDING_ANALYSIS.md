## 2. Pattern Analysis — What Converts

### Universal patterns (all 4 apps):
1. **Feature showcase → Social proof/trust → Paywall** is the dominant flow
2. Every app leads with **"Scan"** as the hero feature
3. **Value/pricing** is always the #2 feature shown
4. All use **dark/premium backgrounds** to feel high-end
5. All end with a **paywall** as the final onboarding screen

### High-conversion patterns (best practices):
| Pattern | Used by | Conversion impact |
|---------|---------|-------------------|
| "Try for $0.00" / free trial framing | App 1, 2 | ⬆️ Trial start rate +30-50% |
| Trial timeline visualization | App 2 | ⬆️ Reduces cancellation anxiety |
| "No Payment Due Now" reassurance | App 2 | ⬆️ Overcomes payment objection |
| Social proof before paywall | App 1, 3 | ⬆️ Trust → higher conversion |
| Micro-pricing ("$0.58/week") | App 3 | ⬆️ Price anchoring |
| "SAVE X%" badge on annual | App 1, 3, 4 | ⬆️ Annual plan selection |
| Curiosity/mystery element | App 4 | ⬆️ Engagement, reduces skip |
| Urgency ("X joined today!") | App 3 | ⬆️ FOMO effect |
| Short funnel (≤5 screens) | App 2, 3, 4 | ⬆️ Less drop-off |

### Anti-patterns (avoid):
| Pattern | Problem |
|---------|---------|
| Rating ask during onboarding | User hasn't used app — feels pushy, wastes a screen |
| No trial offer | Major conversion killer for $30+/yr apps |
| Generic copy ("valuable insights") | Doesn't differentiate from competitors |
| Too many screens (6+) | Each screen loses ~15-20% of users |

## 4. Recommended Onboarding Flow — "The Converter"

### Design philosophy:
- **5 screens max** (each screen loses 15-20% of users)
- **Show, don't tell** — visual demos > bullet points
- **Trust before money** — social proof primes the paywall
- **Free trial framing** — "$0.00" converts better than "free trial"
- **Remove rating ask** — earn it after first successful scan, not before

### Flow:

```
Screen 1 → Screen 2 → Screen 3 → Screen 4 → Screen 5
 HOOK      VALUE       TRUST      CAMERA      PAYWALL
```

---

**Layout:**
- Top 60%: Full-bleed hero image

## 7. Key Decisions & Rationale

| Decision | Rationale |
|----------|-----------|
| **Remove rating ask from onboarding** | User hasn't used app. Trigger after 3rd successful scan instead (higher conversion, better reviews) |
| **5 screens not 6** | Each screen loses 15-20%. 5 is the sweet spot for education + conversion |
| **Social proof before paywall** | Primes trust. Users who see reviews convert 25-40% better on paywall |
| **"Try for $0.00" CTA** | Outperforms "Start free trial" by 20-30% in A/B tests across subscription apps |
| **Trial timeline visualization** | Reduces cancellation fear, which is the #1 objection to starting trials |
| **Annual pre-selected** | 3-5x higher LTV than weekly. Save badge justifies the selection |
| **No sign-in wall** | Friction killer. Let users sign in later from Settings |
| **Camera permission on Screen 4** | After value is established, not before. Higher grant rate |
